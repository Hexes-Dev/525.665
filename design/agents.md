## Rules of Engagement
Adherence to the following standards shall be followed while working on this repository.
1. The primary reference for the project goals, approach and system design is contained in `./DESIGN_REFERENCE.md`. This should be consulted for geneeral guidance on how the project will be approached. Actions that will deviate from this reference should be explicitly requested and the `./DESIGN_REFERENCE.md` file should be updated accordingly. It is the responsibility of the agent making those changes to update the file to match the correct project structure and approach.
2. Major actions will be recorded in the `./action_log.txt` file. This will be refered to prior to making changes to understand what has already been complteded.
3. Execution of python scripts should always be requested to allow for review of the file before it is run.
4. It is key to review how data structures are stored an accessed prior to working on the project. Look through the `src/data/data_tools.py` file to familiarize yourself before you get started.
5. If you're unsure of an action. Please ask for clarification. Always explain the problem in detail and the implications of the choices that might be made.
6. Comments within the code base should be concise and clear explainations of what the code is doing. They should not include numbers outlining the steps of a process.
